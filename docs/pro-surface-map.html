8 · 边框 / 阴影也会制造色差
区域 01

Windows 窗口框

控制入口
一致分组
处理建议

可以一致，不等于应该全部一致。

A、B、C 三组分别承担“连续壁纸画布”“密集导航”“高可读控制面”。每组内部可以完全一致；跨组强行一致会牺牲结构或可读性。

区域 技术上可控 应该和谁一致 不一致的真实原因 目标
Windows 1px 外框 / 系统按钮 部分 OS 组 系统负责最终绘制，主题只能影响承载它的标题栏底色。 视觉接近，不承诺像素一致
左栏顶部 + 左侧导航 可以 B 组内部一致 两个 DOM 表面分别叠色，并夹着边框。 同一侧栏令牌、各只盖一层
主区顶栏 + 正文工作区 可以 A 组内部一致 同一半透明令牌在父层、工具栏、主区重复叠加，最终颜色仍不同。 共用一张连续遮罩，移除分隔线和环境阴影
右侧工具面板 可以 C 组强表面 面板本来就使用更高不透明度，打开时自然出现明显断层。 与输入区/菜单统一；不要与正文强行同色
输入区 / 菜单 / 弹层 可以 C 组内部一致 为可读性主动使用强表面，不是渲染错误。 保留层级，但用同一强表面令牌
边框 / 阴影 / backdrop-filter 可以 跟随所属表面 1px 浅线、暗影和模糊会让相同底色看起来不同。 连续区域归零；交互控件才保留

根治标准：同一连续平面只允许一个半透明背景拥有者。不能只把几个 CSS 颜色值写成相同，因为半透明层的最终颜色取决于叠加次数、下方壁纸像素、边框、阴影与模糊。先收敛“谁负责上色”，再统一令牌。
